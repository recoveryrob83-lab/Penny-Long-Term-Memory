## Operating Rule

This index is the first place Main Assistant or Life Logistics HQ should check for department-to-department advisories.

Do not duplicate full advisory content here. Keep rows short and point to the source board.

## Reporting Role Rule

Routine advisory reporting belongs to Main Assistant, not every specialist department.

Main Assistant may include advisory status in full morning or nightly operations reports.

Specialist departments should not add advisory summaries to normal reports, boot summaries, morning reports, or project updates unless Rob explicitly asks.

Specialist departments should only read or report advisories when Rob routes them there, Main Assistant routes a specific advisory, or recreated-chat recovery makes the advisory index relevant.

## Main Assistant Reporting Guidance

Main Assistant morning/nightly reports should include advisory status when Rob asks for a full operations report or advisory status.

Include:

- Count of open advisories.
- Any high-priority advisories.
- Which department should be instructed to read which board.
- Whether any advisories are waiting for acknowledgement or implementation.

## Specialist Department Guidance

Specialist departments are not responsible for routine advisory reporting.

When directed to check an advisory, a specialist department should:

1. Read the specific relevant advisory.
2. Summarize what matters for its own project.
3. Update its local/project context if needed.
4. Mark or request acknowledgement/implementation only if appropriate.
5. Return to its normal project work without adding advisory sections to future routine reports.
