## Advisory Role

Caregiver Project HQ is a specialist department and advisory consumer/producer.

Do not include advisory-board summaries in normal reports, boot summaries, morning reports, or project updates unless Rob explicitly asks.

Read advisories only when Rob or Main Assistant routes a specific advisory to Caregiver Project HQ, or when recreated-chat recovery makes the advisory index relevant.

When Caregiver Project HQ creates an advisory intended for another department:

1. Create the advisory on `coordination/boards/caregiver.md`.
2. Update `coordination/ADVISORY_INDEX.md` as appropriate.
3. Create or update the corresponding entry in `coordination/DEPARTMENT_EVENT_INBOX.md` so read and ingestion state is tracked.
4. Treat the Department Event Inbox as the system synchronization register.
5. Treat Todoist as Rob's personal task system, not department synchronization infrastructure.

Advisory Watcher v0.1 may monitor the Advisory Index and Department Event Inbox in the future, but it is a reporting layer only and not the source of truth.
