## Purpose

This folder stores durable Engineering notebook leaf notes.

Notebook leaves are for ideas, observations, lessons learned, architecture reflections, and useful discussion fragments that may later be promoted into advisories, design docs, implementation packets, tasks, or operating rules.

## Naming Convention

Use:

`NOTE-YYYYMMDD-###-short-slug.md`

Example:

`NOTE-20260705-001-scheduled-task-connector-behavior.md`

## Capture Rules

- Keep notes abstract and non-sensitive.
- Do not store secrets, credentials, API keys, tokens, or private operational details.
- Notebook leaves are not advisory boards.
- Notebook leaves are not task lists.
- Notebook leaves are not authoritative source-of-truth files.
- Promote a notebook note intentionally when it becomes a formal advisory, decision rule, design doc, implementation packet, or Rob-facing task.
